# 🚇 Smart Metro Ticketing & Analytics System

## Overview

This project implements a **Smart Metro Ticketing System** with
integrated **AI-powered analytics** for demand prediction and fraud
detection.

The system consists of three major components:

1.  Backend API Server -- Node.js + Express
2.  AI Microservice -- Python FastAPI
3.  Database -- SQLite

Frontend integration will be implemented next.

------------------------------------------------------------------------

# 🏗 System Architecture

Frontend (Coming Soon) ↓ Node.js Express Backend ↓ SQLite Database ↓ AI
Microservice (FastAPI)

Backend handles ticketing logic and communicates with the AI service for
predictions and fraud analysis.

# 🗄 Database Schema

## Stations

Stores metro station details.

Table: `stations`

Columns: - id - code - name - line

------------------------------------------------------------------------

## Metro Network

Stores station sequence and distance between consecutive stations.

Table: `metro_network`

Columns: - station_code - next_station_code - line - sequence -
is_interchange - latitude - longitude - distance_to_next_km

------------------------------------------------------------------------

## Tickets

Stores booked ticket information.

Table: `tickets`

Columns: - id - source_station - destination_station - booking_time -
travel_time - crowd_level - qr_code

------------------------------------------------------------------------

## Ticket Validations

Stores entry and exit validations.

Table: `ticket_validations`

Columns: - ticket_id - entry_station - exit_station - entry_time -
exit_time - travel_time - distance

------------------------------------------------------------------------

## Fraud Alerts

Stores suspicious ticket usage detected by AI.

Table: `fraud_alerts`

Columns: - ticket_id - fraud_probability - reason - created_at

------------------------------------------------------------------------

## Predictions

Stores demand predictions generated by the AI model.

Table: `predictions`

Columns: - station - hour - day - predicted_demand - crowd_level -
created_at

------------------------------------------------------------------------

# 🔌 Backend APIs

Base URL

/api

------------------------------------------------------------------------

## 1. Health Check

Endpoint:

GET /health

Response:

{ "status": "Backend running" }

------------------------------------------------------------------------

## 2. Stations API

Get all stations

GET /api/stations

Returns a list of metro stations.

------------------------------------------------------------------------

## 3. Book Ticket

Endpoint:

POST /api/tickets/book

Request Example:

{ "source": "Biocon Hebbagodi", "destination": "Central Silk Board",
"time": "09:30" }

Process: 1. Validate stations are on same line 2. Call AI demand
prediction 3. Generate QR ticket 4. Store ticket in database 5. Store
demand prediction

------------------------------------------------------------------------

## 4. Validate Ticket

Endpoint:

POST /api/tickets/validate

Request:

{ "ticket_id": "T123456" }

Response:

{ "status": "valid" }

------------------------------------------------------------------------

## 5. Entry Scan

Endpoint:

POST /api/tickets/entry

Request:

{ "ticket_id": "T123456", "entry_station": "Biocon Hebbagodi" }

Response:

{ "status": "entry recorded" }

------------------------------------------------------------------------

## 6. Exit Scan

Endpoint:

POST /api/tickets/exit

Request:

{ "ticket_id": "T123456", "exit_station": "Central Silk Board" }

Process: 1. Validate ticket 2. Check entry record 3. Calculate travel
time 4. Calculate distance using metro network 5. Run AI fraud detection
6. Save fraud alerts 7. Update ticket validation

Response Example:

{ "status": "exit recorded", "distance_km": 11.9, "travel_time_minutes":
20, "fraud_analysis": { "fraud_probability": 0.12, "alert": false } }

------------------------------------------------------------------------

## 7. Fraud Alerts

Get all fraud alerts

GET /api/fraud-alerts

Used for admin monitoring dashboard.

Manual fraud check

POST /api/fraud-alerts/check

------------------------------------------------------------------------

## 8. Demand Predictions

GET /api/predictions

Returns stored demand predictions for analytics.

------------------------------------------------------------------------

# 🤖 AI Microservice APIs

## Demand Prediction

POST /predict-demand

Input:

{ "station": "Biocon Hebbagodi", "hour": 9, "day": "Tuesday", "weather":
"Clear", "event": 0 }

Output:

{ "predicted_demand": 12980, "crowd_level": "Medium" }

Accuracy: \~91%

------------------------------------------------------------------------

## Fraud Detection

POST /detect-fraud

Input:

{ "entry_station": "...", "exit_station": "...", "entry_hour": 9,
"travel_time": 20, "ticket_type": "QR", "distance": 11.9 }

Output:

{ "fraud_probability": 0.93, "alert": true, "reason": "Unusually long
travel duration" }

Accuracy: \~99%
